How does the internet work? (expanded)

Networks [&] protocols

  • When surfing the internet, your data usually goes through a ISP (Internet Service Provider)!
  • ISPs then send your request to differnet servers and those servers get back to you

Protocols are how data is transmitted and received between devices, and they include but are not limited to: HTTPS, TCP/IP and many more they work by one device that sends a request using a protocol such as HTTPS for a website, and the data is broken into packets, which is recieved by the second device. Physical connections also play a big part in the internet, such as fiber optic cables, and underground internet cables, which link continents together online. they are a vital part of the internet, as if we didn't have them, we wouldn't have any internet.

Packet switching [&] Routing

  • Packet switching: Data broken into smaller packets, sent and reassembled
  • Routing: the best path for these packets to travel
  • Reliability: makes sure that all data is recieved, if not, they are reordered

What is Encoding?

Encoding is the process of converting data from one form to another, mainly for processing, storage or transmission. there are many ways of encoding such as:

  • Binary: 2 digit system, 1s and 0s
  • - 8 bits of binary form a byte, and scale up to KB, MB, GB and so on
  • ASCII: encoding language that represents text using numbers
  • - Uses 7 bits to show 128 characters, from digits to letters, etc

What is Encryption?

Encryption is a crucial part of the internet, as it helps protect crucial data when transmitting. It revolves around using ciphers or other ways of encryption to keep sensitive files or texts safe when sent across the web.

  • Symmetric Encryption: uses 1 key to encrypt and decrypt data, making it faster [&] more efficient
  • Asymmetric Encryption: uses 2 keys, private key [&] public key
  • -Public key used to encrypt and send data, private key used to decrypt once recieved

HTTPS [&] Security

HTTPS uses TLS (Transport Layer Security) to basically encrypt data that is sent through a client's search engine and the server. And the browser does a TLS handshake with the server when the client enters a website, which involves using encryption keys to verify their identity.
